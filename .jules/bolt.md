## 2025-06-20 - Fast TT Probing Avoids Maybe Allocation
**Learning:** In the hot path of search (AlphaBeta and Quiescence), allocating a `Maybe` wrapper (and a boxed tuple inside it) for Transposition Table lookups via `probeTT` causes significant GC pressure. This is a common anti-pattern where a boxed structure is created only to be immediately unpacked.
**Action:** Replaced `probeTT` with `probeTTFast` in hot search loops. `probeTTFast` simply returns the raw `(Word64, Word64)` key-data pair unboxed, and caller checks `key == hash` and directly calls `unpackData`, eliminating all intermediate `Maybe` and tuple allocations during TT hits and misses. Ensure to use strict tuple returns or single tuple unpacks directly on the result.
