## 2024-05-24 – [Perft Leaf Node Move Counting Optimization]
**Learning:** `perftWhite` and `perftBlack` functions in `Chess.Core.Rules.Class` were evaluating `length (generateMoves game)` for leaf nodes at `depth == 1`. In Standard chess variant, `generateMoves` constructs the full list of moves before computing its length, causing an $O(N)$ allocation of list nodes and boxed moves.
**Action:** Introduced a `countMoves` method to the `ChessVariant` typeclass to bypass generating the full list where possible. Implemented `countMoves` for `Standard` variant to directly return the number of legal moves using `countLegalGenMovesSafe` (or `countLegalGenMoves` when in check), bypassing list allocation entirely for depth 1 leaf node counting.
**Impact:** Minor allocation reduction. On `bench-core` KiwiPete Depth 4: NPS increased slightly.
## 2024-05-24 - [Unboxed Tuple Alternative in TT Lookups]
**Learning:** Returning an unboxed tuple from an `IO` action like `probeTTFast` (e.g., `IO (# Word64, Word64 #)`) requires special handling and causes kind mismatches (`Expected kind '*', but '(# Word64, Word64 #)' has kind 'TYPE'`) under normal signatures without explicit unwrapping. However, returning a standard tuple `(Word64, Word64)` combined with a strict `{-# INLINE #-}` pragma is successfully unboxed by GHC in the hot path, achieving the same allocation-free behavior. Using `Maybe` in the core TT probe loop results in unnecessary heap allocations (boxing). By unconditionally returning the raw TT entry key and data as an unboxed tuple and having the caller perform the `entryKey == hash` check and explicit unpack via `unpackDataFast`, we drastically reduce GC pressure and allocations during deep searches.
**Action:** In Haskell hot paths (like Transposition Table lookups), avoid returning or unpacking into multi-element tuples wrapped in `Maybe`, as this forces heap allocation (boxing). Keep data bit-packed as unboxed values (like `Word64`) and return them as an inlined tuple `(Word64, Word64)`. Let the caller verify the cache hit directly to prevent boxed value allocations.
